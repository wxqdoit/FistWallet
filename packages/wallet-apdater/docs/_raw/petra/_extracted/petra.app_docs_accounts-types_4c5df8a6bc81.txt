[H1] Account Types
[CODE] Petra supports a number of different account types, including: Secp256k1

[H2] Ed25519
[P] Majority of accounts that are created on Petra are Ed25519 accounts. These accounts are those that were created using:
[LI] A private key
[LI] A secret recovery phrase
[LI] A hardware wallet (e.g. Ledger, Arculus, etc.)
[CODE] When you sign a message, request a connection, or sign a transaction, the wallet will return Ed25519 instances of the Signature
[CODE] PublicKey
[CODE] AccountAuthenticator

[H2] Keyless
[P] Some accounts may be created as a Keyless account which are created using social logins:
[LI] Sign in with Google
[LI] Sign in with Apple
[P] These accounts are represented as single key accounts which are use generalized authentication scheme (opens in a new tab) supporting functions schemes like Keyless.
[CODE] When you sign a message, request a connection, or sign a transaction, the wallet will return single key instances of the Signature

[H2] Secp256k1
[P] A small number of accounts are created as Secp256k1 accounts which are created using a private key.
[P] Likewise to Keyless, Secp256k1 accounts are represented as single key accounts which are use generalized authentication scheme (opens in a new tab).
